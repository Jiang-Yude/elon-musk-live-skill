# 正式技能包更新 Gate

每日資料不等於正式技能包更新。

活人每天都會發言，新聞也每天都會解讀。若每一則材料都改 `SKILL.md`，技能包會被短期噪音污染。

## Gate 1：來源可信度

以下任一條成立才進下一關：

- 來自 `musk_direct`
- 來自 `company_primary`
- 來自兩個以上可靠 `news_report`
- 來自一個 `news_report`，且可連回原始訪談、文件或貼文

純 `news_commentary` 不進正式技能包，只能進「外界觀察」或「批評材料」。

## Gate 2：重複性

以下任一條成立才進下一關：

- 同一觀點跨至少兩個不同場景重複出現
- 同一判斷有公開行動印證
- 與舊觀點明顯衝突，且本人或公司後續補充

單次玩笑、meme、情緒反應，不進正式模型。

## Gate 3：結構影響

至少影響一個正式段落：

- 核心心智模型
- 決策啟發式
- 表達 DNA
- 反模式與局限
- 人物時間線
- 最新動態
- 誠實邊界

如果只能當新聞摘要，就留在 daily 或 weekly。

## Gate 4：替換成本

提出更新時要說清楚：

- 要改哪一段
- 新材料補了什麼
- 舊段落哪裡不足
- 是否需要刪除或濃縮舊內容

找不到該改哪段，就不改 `SKILL.md`。

## 建議節奏

- 每日：自動抓資料，產 daily digest
- 每週：人工或 agent 審查 review queue
- 每月：必要時更新正式 `SKILL.md` 與備份

## 輸出格式

正式更新候選用這個格式：

```markdown
## 更新候選：一句話標題

來源等級：
來源：
日期：
影響段落：
建議改法：
舊段落不足：
不確定點：
是否需要 Claude 補審：
```
