# 來源分級規則

## 五級來源

### 1. `musk_direct`

馬斯克本人直接產出的公開內容。

包含：

- X 貼文與回覆
- 長訪談與播客逐字稿
- 法庭證詞
- 股東會、產品發表、公司活動中的直接發言

用途：

- 支撐「他說過什麼」
- 觀察措辭、語氣、重複主題
- 判斷觀點是否演化

限制：

- 貼文可能是玩笑、反諷、情緒反應或市場操作
- 單則貼文不得直接升級成心智模型

### 2. `musk_direct_public_web`

從公開 X 個人頁 HTML 擷取到的本人公開貼文。

用途：

- 在沒有 `X_BEARER_TOKEN` 時，保留免費公開頁面可讀到的本人公開貼文線索
- 作為每日追蹤的高優先候選
- 補足新聞 RSS 可能漏掉的本人原始說法

限制：

- 這不是官方 API，可能受 X 前端頁面結構、登入牆、地區、快取與反爬策略影響
- 個人頁可能混入置頂、精選或舊貼文，因此腳本只收近期資料
- 若與官方 API、本人訪談或其他來源衝突，以可驗證的一手來源優先

### 3. `company_primary`

他直接控制或高度相關公司的官方來源。

包含：

- Tesla、SpaceX、xAI、Neuralink、X 官方公告
- SEC 文件
- 股東信、財報電話會議逐字稿
- 官方產品文件

用途：

- 支撐「公司做了什麼」
- 檢查公開說法和實際行動是否一致

限制：

- 公司文件不等於馬斯克個人完整想法
- 公關文件需要和行動資料交叉看

### 4. `news_report`

媒體報導的可核對事實。

用途：

- 補足事件時間線
- 找訪談、文件、公開行動的入口

限制：

- 必須保留媒體來源
- 不直接當作本人立場

### 5. `news_commentary`

評論、社論、市場分析、人物解讀。

用途：

- 收集外界如何理解與批評
- 作為反例、盲點、風險觀察

限制：

- 只能寫成「評論者認為」
- 不可寫成事實或本人觀點

## 答案引用規則

回答最新事件時，至少說清楚：

- 這是本人直接說法、公司一手資料、媒體報導，還是評論
- 來源日期
- 是否已被第二來源支持
- 是否進入正式模型，或只留在每日摘要
